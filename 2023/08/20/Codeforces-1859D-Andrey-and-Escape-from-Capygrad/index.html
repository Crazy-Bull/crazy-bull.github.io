Codeforces 1859D Andrey and Escape from Capygrad

Posted on 2023-08-20 Edited on 2023-08-25 In OI

题意

    有n个传送门，每个传送门有四个参数 l <= a <= b <= r，表示当你的位置在 [l, r] 上时，可以选择传送到 [a, b] 上的任意一个点。
    随后有 q 个询问，每个询问给出一个坐标 x 代表你的起始位置，对于每个输入 x，输出通过传送门能最右到达的位置。

数据范围

1 <= n <= 2⋅105
1 <= l <= a <= b <= r <= 109
1 <= q <= 2⋅105
1 <= q <= 109

题解

首先我们考虑在一个特定位置上时我们会如何行动
如果我们在一个没有传送门的位置，我们本身无法行动
如果我们在一个有传送门的位置，我们首先证明：我们无需考虑向左传送

Proof:
反证法：如果最佳路径种包含由点 x1 传送到 x2 ，其中 x2 < x1 ，由于题目中要求终点尽可能在右侧，则一定有终点 d > x1 （否则可以不经过后续路径而停留在 x1 ）
设依次经过的传送门分别为 p1, p2, …, pm
Definition: 如果两条线段 [l1, r1], [l2, r2] 满足 l1 ≤ r2 或 l2 ≤ r1 ，称它们是 “相交的”
不难发现，pi 与 pi+1 对应的线段一定是相交的（由 [a, b] 包含于 [l, r] 保证）
从而这些传送门对应 [l, r] 线段的并集恰为区间 [min{pi.l}, max{pi.r}]
不难发现， min{pi.l} <= x2 < x1 < d <= max{pi.r}，即 x1 在区间内
从而存在一个 pi 使得 x1 在其对应的 [l, r] 线段内，则无需经过 x2 而直接通过此传送门传送即可

推论: 每次传送一定会传送至该传送门的 b 处（否则等效为先传送至 b 再传送至 b 点左侧的某点，由上文知矛盾）

这样我们发现，由于 a 在 b 的左侧， r 在 b 的右侧，故传送门的 a 和 r 参数是无用的
因此我们可以用 [l, b] 代表一个传送门的效果：当你的位置在 [l, b) 上时，一定可以（无副作用地）传送至 b 处
容易想到，当两个传送门的 [l, b] 线段相交时，可以合并它们的线段，并用一个大线段替代其效果
合并的方法为：将线段按 l 关键字排序，若后一个 l 大于前一个（合并后）的线段，则后续所有线段与前面的线段都不相交，因此新开一个线段。否则将该线段与后者合并。
预处理完毕线段后，对每次查询，用二分查找找到其所属的（唯一）大线段，输出对应的 b 值即可。

复杂度：O(nlogn+qlogn)
